Project Name- Order Products Analysis 

In this project, we use a number of SQL queries to answer various questions about a set of data, with a focus being on performing 
joins using our three tables and a number of filters. 

The dataset we are working with is called "SQL Practice Dataset 1 (Easy) + Queries" and can be found on can be found on "https://www.kaggle.com/".

More specifically, it can be found via this address to reach the dataset directly from it's original source- "https://www.kaggle.com/datasets/nudratabbas/sql-practice-dataset-1-easy-queries/data".

Alternativly, a copy of the three data tables we are working with can be found attached to his repository, within the folder "Data Tables". 

To run our work, simply open the file "Order-Products Analysis.db" via DB Browser for SQLite.

SQL logic used-

1. SELECT statements to return information from the table, directly by name or using an alias
2. WHERE clause to filter rows via specified criteria to narrow results down
3. GROUP BY to group results so they can be aggregated over (via aggregates such as COUNT, SUM, AVG)
4. Using scalar functions to adjust data results (such as ROUND)
5. CASE WHEN to adjust/filter results
6. ORDER BY to order resulting table in a different orientation
7. Using joins to connect 2-3 tables together in one query
8. Using CASE WHEN to filter multiple columns in a single SELECT statement
9. Use of CTEs ('Common Table Expression') to filter and aggregate data

Some questions answered via our queries- 
1. Product categories purchased most often by age
2. Count of products sold by age
3. Report of Sales and Revenue by Major Holidays
4. Total Amount of Customers by City Whom Ordered at Least 3 Items from the Beauty Category
5. Amount of Products Sold by Age
6. And more
